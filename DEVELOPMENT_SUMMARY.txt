--------------------------------------------------------------------------------
1. PROJECT OVERVIEW
--------------------------------------------------------------------------------

An NSE (Indian equities) swing trading research platform built in Python.

Core capabilities:
  - Extract trading strategies from YouTube videos using an LLM (Groq)
  - Backtest strategies against historical NSE market data (NIFTY 50 universe)
  - Manage trade records in an Excel workbook
  - Visualise results via a Streamlit dashboard

Technology stack:
  - Language:    Python 3.10+
  - LLM:         Groq API (llama-3.3-70b-versatile)
  - Broker/Data: Zerodha Kite Connect API
  - Fallback:    NSE Bhavcopy CSV archives (free, public)
  - Dashboard:   Streamlit + Plotly
  - Storage:     Excel (openpyxl + xlwings)

How to run:
  bash setup.sh                        # validate env, install deps
  bash run.sh dashboard                # launch dashboard (http://localhost:8501)
  bash run.sh token                    # generate Zerodha OAuth token
  python main.py scrape --url <URL>    # extract strategy from YouTube
  python main.py status                # show open trades from Excel

G. NSE BHAVCOPY DATA LIBRARY
------------------------------
Problem: files downloaded on demand, NSE changed URL format in Jan 2024.
Old dates (<=2023) use a different archive URL pattern.

What we built:
  - get_cached_dates()    : scans cache for both old and new filename formats
  - get_known_holidays()  : reads _nse_holidays.json (confirmed 404 dates)
  - _record_holiday()     : writes confirmed NSE holidays to JSON cache
  - get_coverage_stats()  : cached days + confirmed holidays + missing dates
                            all from one consistent calculation
  - download_bulk()       : downloads missing dates with progress callback,
                            records holidays automatically
  - Dual URL format:
      New (2024+): .../content/cm/BhavCopy_NSE_CM_0_0_0_YYYYMMDD_F_0000.csv.zip
      Old (<=2023): .../content/historical/EQUITIES/YYYY/MMM/cmDDMMMYYYYbhav.csv.zip
  - Old format EQ filter  : filters to EQ series rows only
  - Retry logic           : 3 attempts with exponential backoff (2s, 4s)
  - Timeout               : increased 30s -> 60s

Sidebar 📦 Data Library:
  - X / total days with trading days + holidays breakdown
  - Progress bar (🟢 >=99%, 🟡 >=60%, 🔴 <60%)
  - Date range covered, expandable missing dates list
  - ⬇️ Download missing data button with live progress
  - Coverage % always derived from accounted/total (no stale cached formula)

5. HOW THE BACKTEST WORKS
--------------------------------------------------------------------------------

Data flow:
  1. Select strategy → loads StrategyConfig preset
  2. Set date range / capital / risk %
  3. Click ▶️ Run Backtest
  4. For each of 50 NIFTY symbols:
       - Fetch daily OHLCV (Bhavcopy or Zerodha), cached as CSV
       - Compute 30+ indicators (EMA, MACD, RSI, RSI-2, Bollinger, ATR,
         volume ratios, momentum scores, candlestick patterns)
  5. For each symbol walk daily candles:
       - Evaluate 7 checks (trend, entry_point, volume, MACD, EMA, support, reversal)
       - If score >= minimum_confidence and entry check passes → open trade
       - Exit on: stop-loss hit, target hit, or max holding days elapsed
  6. Sort all trades by (entry_date, score)
  7. Apply max open positions cap
  8. Build equity curve (running P&L)
  9. Display results + auto-run comparison across all 6 strategies

Position sizing:
  quantity = floor((capital × risk_pct / 100) / (entry_price − stop_loss))
